# 需求-解决复合问题

## 需求描述
如果我分析一个APP，这个APP可能既需要分析二进制，又需要网络抓包，那么这就是一个综合的事情，我切换到具体的agent是很难完成一个复合任务的。所以我现在的想法是，创建一个任务分发Agent，根据任务目标、子任务目标，自动调用专业的Agent。


## 难点

- 每个Agent在最开始运行的时候要创建任务目录，如果是一个复合任务我希望所有子Agent使用同一个父任务目录，子Agent可以在父目录创建子Agent特有的目录。但是这会对当前binary-analysis、mobile-analysis、web-analysis这些agent的任务目录的创建有些冲突吧？

是否还有其他难点你帮我找出来。

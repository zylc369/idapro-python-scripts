# 进化-分析持续性增强

安全分析要么是逆向、要么是找漏洞，所以通常比较难，要消耗很长时间。我使用ai-security-analysis、binary-analysis、mobile-analysis、web-analysis、security-coordinator分析安全需求的时候，在执行几十分钟之后通常会自动停止，然后告诉我一些候选的方案、或者咨询我的经验或意见，这并不是我期望看到的，我希望的是如果分析不出来就要一直分析不要问我。

当前是不是已经在提示词中指定了必须一直分析到完成？如果是的话，我觉得大概率是opencode或者LLM自动中断过长任务。

**我想到的根治的方案是：**
1. Hook OpenCode会话不再执行的Hook点，如果是这些agents那么主动发送：`问题解决了吗？如果没有解决那么请自住继续解决。`（你可以帮我润色这句话）。这样的话就能够自动的恢复执行未完成的工作。发送这句话的前后要写日志，便于我调试。
2. 默认持续工作6小时，如果超过这个时间那么就不要发送这句话了，即，不要再恢复执行未完成的工作。
3. 在开启任务的提示词中如果指定了最大的持续工作时间，那么将这个时间写到任务目录下的文件中。
4. 第1步发送之前，要先获取最大持续工作时间，如果获取到则以获取到的为准，否则默认最大6小时。
5. 如果决策步发送第1步的提示词，那么也要打印日志，便于我调试。
6. 这些agents可能会调用**Task 工具**创建子Agent，无论这个子agent是什么，都不要发送第1步的提示词，只在主agent做恢复动作。
7. security-coordinator agent是编排agent，它分析任务并且调用合适的agent去执行部分任务，和第6点的策略一样无论这个子agent是什么，都不要发送第1步的提示词，只在主agent做恢复动作。

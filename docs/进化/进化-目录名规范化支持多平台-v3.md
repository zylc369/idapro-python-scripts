# 进化-目录名规范化支持多平台-v3

## 目录名规范

`~/bw-ida-pro-analysis`改为`~/bw-security-analysis`，所有读写原文件夹都要改为新文件夹。

## 支持多平台

我接下来想要支持pc端、mobile端、web端的安全分析，我想要将文档、代码放到对应端的目录下，这样能够方便的我对它们的阅读和理解。

### Binary Analysis Agent重命名
**Binary-Analysis**改为**Binary-Analysis-PC**

### 工作空间分开
Binary Analysis Agent里面读写`~/bw-security-analysis/workspace/[任务目录]`改为`~/bw-security-analysis/workspace-pc/[任务目录]`。

### opencode扩展目录分开
`.opencode/binary-analysis`改为`.opencode/security-analysis`，和pc有关的放到`.opencode/security-analysis/pc`目录下，通用的直接放到`.opencode/security-analysis`目录下。


## ida-pro-analysis-evolve命令变更
1. `ida-pro-analysis-evolve`更改为`security-analysis-evolve`。
2. 如果没有指定特定要被进化的Agent，则需要询问用户，而不是在命令中写死要进化的Agent。

## gui-interact命令变更
`gui-interact`改为`gui-interact-pc`。

## 其他目录变更
1. `ida-pro-analysis-requirements`改为`security-analysis-requirements`。
2. `ida-pro-analysis-docs`改为`security-analysis-docs`。

## binary-analysis插件更新

binary-analysis插件更改为security-analysis插件。获取当前agent名字。
